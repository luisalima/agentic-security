# Code of Conduct

## Our Goal

This project aims to be a useful, accurate, and welcoming reference for agentic security. To keep it that way, everyone who participates here — maintainers, contributors, and people who file issues or PRs — agrees to a few simple expectations.

## Expected Behavior

- Be respectful and professional in issues, pull requests, code reviews, and any other project space.
- Assume good intent. Disagreement is fine; hostility is not.
- Critique ideas, code, and arguments — not the people behind them.
- Be patient with newcomers. The whole point of this repo is teaching.
- Credit prior work. If a defense, attack, or argument comes from someone else's research, link the source.

## Unacceptable Behavior

- Personal attacks, insults, harassment, or sustained disruption of discussion.
- Discrimination on the basis of who someone is rather than what they're arguing.
- Publishing someone else's private information without permission.
- Using this project as a platform for unrelated political or commercial campaigns.
- Submitting code, content, or reports made in bad faith — including knowingly false security reports or content meant to embed real exploits in the educational examples.

## Scope

This applies in all project spaces: the GitHub repository (issues, pull requests, discussions, code review), any project chat or mailing list if one is set up later, and any time a contributor is representing the project in public.

## Reporting

If you experience or witness behavior that breaks this Code of Conduct, email **luisa@nubia-labs.com**. Reports are read by the project maintainer.

Include what happened, where (link if possible), and who was involved. You'll get a reply within five business days.

Reports are kept confidential to the extent possible while still allowing the issue to be addressed.

## Enforcement

The project maintainer may take any action they consider proportionate, including:

- A private note explaining what's expected.
- A public note in the relevant thread.
- Temporary or permanent removal of comments, commits, or contributions.
- Banning a contributor from project spaces.

The maintainer also reserves the right to act on behavior outside project spaces if it reasonably affects the safety or trust of people inside them.

## Security Issues vs. Conduct Issues

This document is for *behavior* in the project. For *security vulnerabilities* in the code or examples, follow [SECURITY.md](SECURITY.md) instead — those have a separate disclosure process.
